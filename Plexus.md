
# Общие сведения

**Plexus** — многофункциональное веб-приложение, совмещающее в себе:

- базу знаний,
- учёт и анализ доходов и расходов,
- систему управления проектами.

Ключевая особенность приложения — модуль **проектов**, который позволяет объединять знания, финансовые операции и бюджет в рамках достижения конкретной цели за заданный промежуток времени.

---

# Чем является приложение

## Определения

**Проект** — это многоэтапный процесс, направленный на достижение определённой цели в рамках заданного срока и бюджета.  
Проект может включать:

- связанные заметки из базы знаний,
- планируемые и фактические доходы/расходы,
- бюджет,
- задачи и подзадачи,
- связи с финансовыми транзакциями.
**Скелет приложения** - термин, описывающий написание методов и соответствующей документации без имплементации. Это набросок, который используется, чтобы не тратить время на соблюдения контрактов, решению багов по глупым причинам (конфликт имен, неправильный формат данных etc).

## Plexus — это:

- база знаний;
- приложение для учёта и анализа доходов и расходов;
- приложение для управления проектами.

## Временные рамки

### Первичная оценка

Поскольку одна из основных целей создания данного приложения — демонстрация навыков разработки в рамках pet-проекта потенциальному работодателю, а также закрепление практики работы с NestJS, Node.js, TypeScript и смежным стеком, разработку необходимо выполнить в ограниченные сроки.

**Планируемое время разработки:** 180-200 часов
**Срок завершения:** до 6 мая.

# Стек приложения

## Технологический стек MVP

- backend: NestJS / Node.js / TypeScript
- database: PostgreSQL
- ORM: Prisma
- auth: Keycloak
- frontend: SPA на React
- deploy: VPS, Docker

## Что планируется после MVP

- отдельный сервис для поиска работы соискателями (backend) — ASP.NET
- desktop-клиент — .NET MAUI
- mobile-клиент — #TODO
- full-text search — #TODO

## Что входит в MVP

- backend
- API documentation
- тесты
- SPA
- deploy
## Что не входит в MVP

- desktop client
- mobile client
- full-text search
- advanced analytics
- задачи и подзадачи
- rich graph UI
- collaborative features
